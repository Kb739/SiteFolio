combat

Player class contains a WeaponCollider component which on overlaps with enemy HitBox invokes a OnHit Event inside enemy class. Weapon collider’s extent depends on size of the weapon equipped by player character.

  • After first trigger, weapon collision disabled to assure it won’t trigger multiple times for single hit. Weapon collision is also enabled/disabled using notifies in animation timeline at desired frames to prevent irrelevant triggers.
  • Inside OnHit event (enemy’s event graph), IsHit flag is sets true which then is used by enemy AI’s Behavior Tree.
  • OnHit event takes a reference to player’s current attack as an argument, on basis of which target’s reaction to hit is decided. A Base Attack class carries events (HitEvent, BlockEvent, ParryEvent) & info (damage, vfx, sfx, animations etc.) to determine how the target should react to hit based on its current state.

In Order to defeat enemy in an efficient manner , game enforces the player to parry/deflect. Most of the attacks can be deflected , a little lesser can be blocked and a few ones can be only be dodged.Â 

  • A Parry & Block share the same input. When player inputs block button, it sets parry flag true (if isn’t already in parry) & after some delay we reset it back to false.
  • This delay is parry window. Past this window if input is still kept pressed, a block is performed instead.

Critical Hit :


For every x=N (x : number of deflects , N : some Integer), player gets a critical hit chance. With this, performing a basic attack launches a critical hit instead , which deals heavy damage & resets x to 0.

For every deflect beyond N (keep goin without using critical hit), a 1/N is added to critical damage. Once critical hit has performed, this extra added damage resets back to base crit-damage.

Guard Meter :


Top-Mid of the player HUD displays a Guard-Meter. For every block character performs, this meter fills up with a certain amount as defined. Once the meter is full, it goes bright red .This means a warning that no further block can be performed, otherwise player will take a hit instead.

Deflecting/parrying enemy attacks drains this Guard-meter by the same amount, re-enabling player to use block for defense.

Flawless Bonus :

 

For every successful attack by player, a float variable called flawless_Points [0,90] is incremented by n points. Though as soon as our character receives a hit, flawless_points resets back to zero. The value of n added through an attack is some percentage of total damage dealt by that very attack.


So when some skill gets casted, the skill cooldown is reduced by flawless_points %, consuming whole stacked bonus.
