Skill System

Along with the initialization of our Player object, skills are spawned inside the level as actors (objects in world) and their references are hold inside a map as values with skill-name as keys.

  • Special Skills here is the array of Structures with Name (skill-name), Weapon (weapon to associate skill with), Class (derived from BaseSkill class) as member variables.

When input is provided by player controller, Skill-Info is checked to match the input key with skill’s hot key & the player’s currently equipped weapon with skill’s associated weapon. This checks is performed across the all the skill objects & the skill with correctly matched info gets applied.

Now before actually casting a skill, InitializeSkillCast event is called.

  1. Cast Start sets the player’s current state to Skill & set some flags (as shown in image above) which prevents the player from casting any skill, when character is already in the middle of SkillEvent.
  2. SkillEvent is overriden in each derived class from BaseSkill & takes skill’s current stage as argument. SkillStage is a structure which hold stats such as damage, AOE, cooldown etc. which get updated with the progression.
  3. SkillCasted here is the event-dispatcher which serves to inform skill-icon widget (displayed in player HUD) about skill application, so icon-state/design can be updated in order to notify the player.
  • As the SkillEvent gets finished, EndSkillCast gets called which resets the flags aswell as the character’s state & throws a cooldown timer for the duration set based on skill’s current stage.
  • Also an event-dispatcher SkillCompleted gets called, which informs skill-icon widget associated with that skill, to start refilling in-sync with cooldown timer.
