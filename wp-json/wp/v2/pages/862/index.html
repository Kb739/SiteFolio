A Base class called BP_Attack<\/strong> is the foundation of this combo system. Although this class contains various information for the full execution of a strike, main focus of the combo-system is two variables called Next A<\/span> & Next B<\/span>.<\/p>\t\t\t\t\t\t<\/div>\n\t\t\t\t<\/div>\n\t\t\t\t

  • Image above shows the combo tree structure where these white sphere represent BP_Attack objects\/actors which are placed inside the Persistent level. This tree arrangement is done just for the purpose of visualization.<\/li>
  • A variable called CurrentComboHit\u00a0<\/span> is initialized with Root reference at the Beginplay & used as start point for the Sequence. Next A<\/span> & Next B<\/span> variables hold the references to next Attack in the sequence on input A & B respectively.
    • On Receiving input (C Input variable here) first we check to see if current strike\/attack holds valid Next A or Next B based on input. For e.g. in above combo tree root doesn’t have any valid reference for Next B , which means combo cannot start with input B.<\/li>
    • If our player character is already in mid-animation (CCurrent State<\/span> = Attack<\/span>) , on finding valid next attack, it cannot just start right away interrupting current strike, rather by employing Save Next Attack<\/span> variable this command gets saved for the execution on a certain animation notifier.\u00a0<\/li><\/ul><\/li><\/ul>\t\t\t\t\t\t<\/div>\n\t\t\t\t<\/div>\n\t\t\t\t
