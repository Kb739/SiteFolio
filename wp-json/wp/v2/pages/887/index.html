Widget in UE4 doesn’t support inheritance very well. Only variables & functions are carried over to derived widget class but actual widget setup (design) doesn’t get copied. So as workaround for the Menu system , I created different designs for each derived class inherited from BaseMenu<\/strong> widget class , where by using an enum<\/span> variable a particular design (screen) can be selected for the derived class.<\/p>\t\t\t\t\t\t<\/div>\n\t\t\t\t<\/div>\n\t\t\t\t

    References to all the BaseMenu type objects reside inside a map variable in GameInstance class. When main-menu level loads at the start of game, all these objects (main menu , option menu , audio menu , etc.) are added to viewport but by default all are Inactive & Hidden.<\/p>\t\t\t\t\t\t<\/div>\n\t\t\t\t<\/div>\n\t\t\t\t

    Now as an entry point MainMenu<\/strong> (one of the classes derived from BaseMenu<\/strong>) widget is set unhidden with a transition-In followed by its Activation.     As to serve different purposes, there’re a few button types. Implementation of BaseButton class is similar to BaseMenu class in a context that derived classes from BaseButton requires different button-designs aswell. Although there are quite a few types, focus of this post will be NextButton & BackButton , as these are the ones used to navigate across the menu.<\/p>\t\t\t\t\t\t<\/div>\n\t\t\t\t<\/div>\n\t\t\t\t

    • NextButton & BackButton both are objects of SequenceButton (a class derived from BaseButton) type. They are treated as separate on the basis of an enum<\/span> (next , back) variable, which used to decide whether or not Reset<\/span> the button’s parent widget (currently active) to its default state on transition.<\/li>
    • A SequenceButton<\/strong> contains an enum<\/span> variable called NextMenu<\/span> which used as key to find reference to next menu widget (BaseMenu<\/strong> type object) & Enter<\/span> into it.\u00a0<\/li>
    • Before entering into next menu widget, Sequence Button<\/strong> calls Out <\/span>(BaseMenu<\/strong>‘s member function) on its parent menu (currently active), which sets it hidden followed by deactivation.<\/li><\/ul>\t\t\t\t\t\t<\/div>\n\t\t\t\t<\/div>\n\t\t\t\t
