  • Behavior Tree starts with a service<\/span> which ticks to check if player is valid , then sets it as Target<\/span> variable for AI controller.<\/li>
  • GotHit?<\/span> , CanDodge?<\/span> , Attack?<\/span>, Not_Busy?\u00a0<\/span>are Decorators ( perform condition checks ) separated as nodes\/branches for the Selector ( if child node returns false it moves to next node, otherwise aborts & returns true).
    • Not_Busy?<\/span> here is kind of double-check required for certain situations , which assures that the Enemy character is not busy with some other tasks but actually is in idle state. Then it can proceed with basic locomotion around the Target<\/span>.<\/li>
    • GotHit?<\/span> looks for a flag called OnHit<\/span> , which updated to be true<\/span> through a HitEvent<\/span> ( on attacked by player & gets hit) defined inside Enemy<\/strong> class.<\/li>
    • Similarly , Attack?<\/span> checks for a Attack Command <\/span>flag\u00a0which is updated inside a function called TryAttack<\/span> . This function mainly get called repeatedly from a GroupMonitor <\/strong>class object in a certain interval of time. If Attack Command is true, it proceeds with further task of attack-selection & then executing it on Target.\u00a0<\/span>
