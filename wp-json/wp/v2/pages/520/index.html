Fire Puzzle<\/p>\n\t\t\t\t\t\t\t<\/div>\n\t\t<\/div>\n\t\t\t\t<\/div>\n\t\t\t\t<\/div>\n\t\t\t\t

\n\t\t\t\t
\n\t\t\t\t\t\t\t\t

Player has to lit ‘N’ number of fire-pots placed at certain distant locations. A fire-pot can remain lit for ‘T’ seconds , so next pot has to be ignited before previous one blows out & doing that successfully will retrigger the previous pot for extra T seconds. Ones all ‘N’ pots are burning simultaneously , puzzle is solved.<\/p>\t\t\t\t\t\t<\/div>\n\t\t\t\t<\/div>\n\t\t\t\t\t<\/div>\n\t\t<\/div>\n\t\t\t\t\t\t\t<\/div>\n\t\t<\/section>\n\t\t\t\t

\n\t\t\t\t\t\t\t\t

Each attack\/strike in a Combo is an object of a base class named ‘BP_Attack’ , which consists of all the implementation about the execution and affect of strike. \n\t\t\t\t\t\t\t\t

Skill in this game are tied to weapons that means in order to apply any skill, respective weapon has to be equipped. Each Weapon has four skills associated with it. Combat<\/p>\n\t\t\t\t\t\t\t<\/div>\n\t\t<\/div>\n\t\t\t\t<\/div>\n\t\t\t\t<\/div>\n\t\t\t\t

\n\t\t\t\t
\n\t\t\t\t\t\t\t\t

Combat\u00a0 revolves around Parrying\/Deflecting enemy attacks , with some typical blocking & dodging. Parrying certain attacks provides player with time window for assured Hit , aswell as chances for ‘critical hit’ leading to greater damage. There’s no lock-on , rather f(i) i.e. “function of input” is implemented in order to auto-handle it for the player.<\/p>\t\t\t\t\t\t<\/div>\n\t\t\t\t<\/div>\n\t\t\t\t\t<\/div>\n\t\t<\/div>\n\t\t\t\t\t\t\t<\/div>\n\t\t<\/section>\n\t\t\t\t
