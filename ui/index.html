User Interface

Widget in UE4 doesn’t support inheritance very well. Only variables & functions are carried over to derived widget class but actual widget setup (design) doesn’t get copied. So as workaround for the Menu system , I created different designs for each derived class inherited from BaseMenu widget class , where by using an enum variable a particular design (screen) can be selected for the derived class.

  • Since a ScreenDesign widget class consists of all the widgets/elements has to be displayed for a BaseMenu class, screen selection has to take place during Pre Construct event ,which serves to provide visualization of elements in order to write events & functions based on these elements.
  • BaseMenu class contains all the Design widgets but as default these all are kept hidden/disabled. Now when a class is derived from it, one specific design is selected & set unhidden for it based on enum.

References to all the BaseMenu type objects reside inside a map variable in GameInstance class. When main-menu level loads at the start of game, all these objects (main menu , option menu , audio menu , etc.) are added to viewport but by default all are Inactive & Hidden.

Now as an entry point MainMenu (one of the classes derived from BaseMenu) widget is set unhidden with a transition-In followed by its Activation. This transition & activation is implemented inside Enter (a member function of BaseMenu class). At this point user can interact with MainMenu.

As to serve different purposes, there’re a few button types. Implementation of BaseButton class is similar to BaseMenu class in a context that derived classes from BaseButton requires different button-designs aswell. Although there are quite a few types, focus of this post will be NextButton & BackButton , as these are the ones used to navigate across the menu.

  • NextButton & BackButton both are objects of SequenceButton (a class derived from BaseButton) type. They are treated as separate on the basis of an enum (next , back) variable, which used to decide whether or not Reset the button’s parent widget (currently active) to its default state on transition.
  • A SequenceButton contains an enum variable called NextMenu which used as key to find reference to next menu widget (BaseMenu type object) & Enter into it. 
  • Before entering into next menu widget, Sequence Button calls Out (BaseMenu‘s member function) on its parent menu (currently active), which sets it hidden followed by deactivation.

As an example , In main menu , NewGame & Exit button can be Event Type, whereas an Option button can be a Sequence type (NextButton) button leading to Options-Menu (BaseMenu type) as the next widget.
