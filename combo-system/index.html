Combo System

A Base class called BP_Attack is the foundation of this combo system. Although this class contains various information for the full execution of a strike, main focus of the combo-system is two variables called Next A & Next B.

  • Image above shows the combo tree structure where these white sphere represent BP_Attack objects/actors which are placed inside the Persistent level. This tree arrangement is done just for the purpose of visualization.
  • A variable called CurrentComboHit  is initialized with Root reference at the Beginplay & used as start point for the Sequence. Next A & Next B variables hold the references to next Attack in the sequence on input A & B respectively.
    • On Receiving input (C Input variable here) first we check to see if current strike/attack holds valid Next A or Next B based on input. For e.g. in above combo tree root doesn’t have any valid reference for Next B , which means combo cannot start with input B.
    • If our player character is already in mid-animation (CCurrent State = Attack) , on finding valid next attack, it cannot just start right away interrupting current strike, rather by employing Save Next Attack variable this command gets saved for the execution on a certain animation notifier. 
  • ResetSavedAttack notify sets the Save Next Attack variable to false , as a result input given in time range O-R for next strike will be dismissed. This serves the purpose to switch animations nearly in-sync with the input i.e. player cannot just press two super-fast consecutive inputs, then wait & expect to execute two consecutive attacks in chain.
  • R-E is the range (after ResetSavedAttack notifier) where next input will be registered. If it was between R-S , Combo will chain from point S(begin of combo window). If input was between S-E, chain will start from very point where input is given as AN_ComboWindow  keeps performing checks for its span.
  • Now in order to execute next strike in chain, AN_ComboWindow calls the ChainCombo event.
  • Set approprate rotation is collapsed node consists of some functions works to adjust facing of character respect to target during strike animation.
  • Inside SetNextHit ,  CurrentComboHit value is replaced with reference to the next attack object based on Input.
